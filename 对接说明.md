# Drone Kalman Filter 对接说明

## 1. 适用范围

这套代码用于对 `DetectionTarget` 消息流做 2D 轨迹平滑。

当前版本只处理：
- `spatial.position.latitude`
- `spatial.position.longitude`

当前版本不会处理：
- `altitude`
- `heightAgl`
- `velocity`
- `orientation`
- 跨 `deviceId` 轨迹融合

核心目标是：
- 输入实时逐条 JSON
- 输出仍然是逐条 JSON
- 只修改经纬度
- 其他字段全部透传

## 2. 部署方式

项目根目录执行：

```bash
python -m pip install -e .
```

安装完成后，主链路入口有两种：
- Python 内直接调用 `DroneKalmanFilterPlugin`
- 命令行调用 `python -m drone_kalman_filter.cli`

## 3. 输入要求

### 3.1 输入形态

主链路支持两种输入方式：
- 实时逐条输入 `dict`
- 离线按行输入 `JSONL`

实时场景下，算法**不要求整段轨迹先到齐**，而是每来一条就处理一条。

### 3.2 最低必需字段

要进入平滑链路，单条消息至少需要这些字段：

```json
{
  "identity": {
    "targetId": "1581F7FVC253U00DBX70",
    "traceId": "0DBX706053"
  },
  "source": {
    "deviceId": "fusion"
  },
  "spatial": {
    "position": {
      "latitude": "23.048578",
      "longitude": "113.090210"
    }
  },
  "eventTime": "2026-03-10T01:41:18.493Z"
}
```

字段作用：
- `identity.targetId`
  - 目标标识
- `source.deviceId`
  - 设备标识
- `spatial.position.latitude/longitude`
  - 平滑主观测
- `eventTime`
  - 计算相邻观测时间间隔
- `identity.traceId`
  - 可用时参与切段；为空时算法仍可运行

### 3.3 不能平滑时的行为

如果单条消息缺少关键字段，或者经纬度无法解析：
- 这条消息不会进入平滑计算
- 会原样透传

## 4. 输出是什么

输出仍然是同结构的 `DetectionTarget` JSON。

当前版本只允许修改：
- `spatial.position.latitude`
- `spatial.position.longitude`

其他字段必须保持输入原值，包括：
- `identity`
- `source`
- `lifecycle`
- `extension`
- `eventTime`
- `processTime`
- `originalData`
- `altitude`
- `heightAgl`
- `velocity`
- `orientation`

也就是说：
- 输入是什么结构
- 输出就还是什么结构
- 只是经纬度会被替换成平滑后的值

## 5. 实时对接方式

### 5.1 Python 直接接流

```python
from drone_kalman_filter import DroneKalmanFilterPlugin, PluginConfig

plugin = DroneKalmanFilterPlugin(
    PluginConfig(
        smoother_mode="robust_prefilter_kalman",
    )
)

# message_dict 是一条 DetectionTarget 消息
outputs = plugin.process(message_dict)
for item in outputs:
    consume(item)

# 流结束、切流、或程序退出前，记得 flush
for item in plugin.flush():
    consume(item)
```

### 5.2 实时接口语义

主链路是固定滞后的小窗口平滑器，所以：
- 输入是一条一条进来
- 输出也是一条一条释放
- 但不保证“输入一条立刻输出一条”

原因是：
- 算法需要少量未来点帮助平滑当前点
- 所以前几个点会暂存在窗口里
- 窗口成熟后再按顺序释放

主接口：
- `process(message: dict) -> list[dict]`
- `process_json_line(line: str) -> list[str]`
- `flush() -> list[dict]`
- `flush_json() -> list[str]`

说明：
- `process(...)` 返回 `list`，因为一次输入可能：
  - 暂时没有成熟输出
  - 或释放一条已经成熟的结果
- `flush()` 用于补齐尾部未释放的点

### 5.3 实时切段

轨迹状态按 `(targetId, deviceId)` 隔离。

当前会触发新段的典型情况：
- `traceId` 变化
- `eventTime` 倒退
- 相邻点时间 gap 过大

这意味着：
- 同一个 `targetId` 在不同 `deviceId` 上，会分别维护各自状态
- 不会做跨设备融合

## 6. 文件批处理方式

### 6.1 生成平滑结果

```bash
python -m drone_kalman_filter.cli smooth ^
  --smoother-mode robust_prefilter_kalman ^
  --input data\core-target-raw.jsonl ^
  --output out\core-target-smoothed-robust.jsonl
```

也可以切到更保守的模式：

```bash
python -m drone_kalman_filter.cli smooth ^
  --smoother-mode kalman ^
  --input data\core-target-raw.jsonl ^
  --output out\core-target-smoothed-kalman.jsonl
```

### 6.2 批处理输出语义

输出文件仍然是一行一条消息的 `JSONL`。

保证：
- 输出条数最终与输入条数一致
- 输出顺序与输入顺序一致
- 除经纬度外其他字段透传

## 7. 选型建议

### `kalman`

特点：
- 链路更简单
- 对一般抖动有效
- 对高频跳变更保守

适合：
- 数据相对干净
- 先求稳接入

### `robust_prefilter_kalman`

特点：
- 先做鲁棒预处理，再进入 Kalman
- 更适合压短时跳变、局部尖刺和来回抽动

适合：
- 前端更关注“轨迹线要稳”
- 原始数据中存在明显跳变

如果当前目标是前端轨迹展示，通常优先试：
- `robust_prefilter_kalman`

## 8. 接入后的最小验收

至少检查这四件事：

1. 输出条数是否与输入条数一致
2. 输出顺序是否与输入顺序一致
3. 除经纬度外其他字段是否完全透传
4. 前端轨迹是否明显减少跳变和来回抽动

如果要做离线检查，可配合：
- `python -m drone_kalman_filter.cli acceptance`
- `python -m drone_kalman_visualization.compare`

但这两者属于验收工具，不是实时接入必需流程。

## 9. 推荐的对接顺序

推荐按下面的顺序落地：

1. 先用 `process(message)` 接单条实时消息
2. 打通输入输出链路
3. 验证输出结构和透传是否正确
4. 再观察前端轨迹是否平滑
5. 最后再根据数据特点调模式或调参数

## 10. 注意

这套代码当前不是：
- 三维轨迹融合器
- 多设备目标融合器
- 离线整段最优重建器

它当前是：
- 一个面向实时流的 2D 轨迹平滑插件
- 面向前端轨迹显示
- 目标是在不破坏消息结构的前提下，让线条更平滑
